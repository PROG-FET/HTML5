  1. Git Basics
    • git init

      • The is the first step in creating a git repo

    • git clone

      • Clon repo located at "repo" onto local machine. Original repo can be located on the local filesystem or on a remote machine via http or ssh

    • git config

      • Define author name to be used for all commits in current repo. Devs commonly use --global flag to set config options for current user

    • git add

      • Stage all changes in "directory" for the next commits.

    • git commit -m

      • Commits the staged snapshot, but instead of launching a text editor, use "message" as the commits message

    • git status

      • List which files are staged, unstaged, and untracked

    • git log

      • Display the entire commit history using the default format.

    • git diff

      • Show unstaged changes between your index and working directory

  2. Undoing changes
    • git revert

      • Create new commit that undoes all of the changes made in "commits", then apply it to the current branch

    • git reset

      • Remove "file" from the staging area, but leave the working directory unchanged. This unstages a file without overwriting any changes.

    • git clean

      • Shows which files would be removed from working directory. Use the -f flag in place of the -n flag to execute the clean

  3. Rewritting Git History
    • git commit

      • commits (saves) the staged snapshot, but instead of launching a text edito, use message as the commit message

    • git rebase

      • Rebase the current branch onto "base" can be a commit ID, a branch name, a tag, or a relative reference to HEAD

    • git reflog

      • Shows a log of changes to the local repository's HEAD. And --relative-date flag to show date info or --all to show all refs.

  4. Git Branches
    • git Branches

      • List all of the branches in your repo. Add a "" argument to" create a new branch with the name "branch"

    • git checkout

      • Create and checkout a new branch named "branch". Drop the -b flag to checkout an existing branch.

    • git merge

      • Merge "branch" into the current branch.

  5. Remote Repositories
    • git remote add |name||url|

      • Creat a new connection to a remote repo. After adding a remote, you can use |name|as a shortcut for |url| in other commands

    • git fetch |remote||branch|

      • Fetches a specific |branch|, from the repo. Leav off |branch| to fetch all remote refs

    • git pull |remote|

      • Fetches the specified remote's copy of current branch and immediately merge it into the local copy

    • git push |remote||branch|

      • Push the branch to |remote|, along with necessary commits and objects. Creates named branch in the remote repo if it doesn't exist.

  6. git config
    • git config --global user.name

      • Define the author name to be used for all commits by the current user.

    • git config --global user.email

      • Define the author email to be used for all commits by the current user.

    • git config --global alias."alias name"

      • Create shortcut for a Git command. E.g. alias.glog log --graph --oneline will set git glog equivalent to git log --graph --oneline.

    • git config --system core.edit "editor"

      • Set text editor used by commands for all users on the machines. |editor| arg should be the command that launches the desired editor.

    • git config --global --edit

      • Open the global configuration file in a text editor for manual editing.

  7. Git Log
    • git log -|limit|

      • Limits number of |limit| to certain number.

    • git log --oneline

      • Condense eacg commit to a single line

    • git log -p

      • Display the full diff of each Commits

    • git log --stat

      • include which files were altered and the relative number of lines that were added or deleted from each of them

    • git log --authors="pattern"

      • Search for commits by a particular author

    • git log --grep="pattern"

      • Search for commits with a commit message that maches |pattern|

    • git log "since" .. "until"

      • Shows commits that occur between "since" and "until". Args can be a commit ID, branch name, , or any other kind of revision reference.

    • git log --graph --decorate

      • --graph flag draws a text based graph of commits on left side of commit msgs. --decorate adds names of branches or tags of commits shown.
